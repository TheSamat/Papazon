При обращении к свойству и полю синтаксис одинаков, однако поле - тип, а свойство - метод. Поэтому имеется разница при их 
использовании на первый взгяд не очевидна:

1) Свойства могут быть доступны как для чтения, так и для записи, но каждое свойство - только для одного или вообще не существовать,
   в то время как поле доступна для чтения и записи одновременно.
2) Свойства будучи методом может вызывать исключения, что поле не способна.
3) Свойства нельзя передовать в метод в качестве параметров с ключевым словом out или ref (свойства возвращают вывод их метода 
   при их применении):
   
    public sealed class SomeType
    {
        private static String Name
        {
            get { return null; }
            set {}
        }
  
        static void MethodWithOutParam(out String n) { n = null; }
  
        public static void Main() 
        {
            // При попытке скомпилировать следующую строку
            // компилятор вернет сообщение об ошибке:
            // error CS0206: A property or indexer may not
            // be passed as an out or ref parameter.
            MethodWithOutParam(out Name);
        }
    }

4) Свойства выполняются медленней обращений к полям. При работе с потоками рекомендуется использовать методы, ибо свойства 
   могут приостановить поток. Кроме того, классы, унаследованные от System.MarshalByRefObject (удаленный доступ к классу), свойства 
   выполняются слишком медленно, поэтому их использование в подобных классах не рекомендуется.
5) При многократных вызовах свойств подряд они могут вызывать поначалу разные значения, а после одно и то же. System.DateTime
   неизменяемое свойство Now - практичный пример.
6) Свойство-метод может порождать видимые побочные эффекты, невозможные при доступе к полю. Так порядок определения значений 
   свойств типа может повлиять на поведение типа.
7) Свойству-методу может затребоваться допольнительная память или ссылка на обьект, не являющийся частью состояния обьекта, так 
   изменение возвращаемого обьекта никак не влияет на исходный, в то время как при обращению к полю всегда возвращается ссылка 
   гарантированно на обьект. Свойство, возвращающее копию - часто возникающая проблема, и еще чаще - не упомянутая в документациях.

Итак, единственная привлекательная черта свойств — упрощенный синтаксис, все остальное — недостатки, в числе которых потеря в 
производительности и читабельности кода.

От CODE_GLOG: при отладке в Visual Studio свойства и неявные методы автоматически обновляются при переходе по каждой строчке, что может
убить сервак, трафик или операционку, в зависимости от реализации свойства. Чтобы это отключить надо зайти в:
    Средства/Параметры/Отладка/Общее/галочка "Включить вычисление свойств и других неявных вызовов методов"
