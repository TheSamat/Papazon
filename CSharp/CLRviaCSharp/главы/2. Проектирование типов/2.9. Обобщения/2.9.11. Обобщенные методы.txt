Методы в отличии других членов обобщенного может иметь дополнительно свои обобщенные параметры помимо параметра типа:
    class Test<T>   // T - обобщенный параметр типа 
    {
        void Print<Tcw>(T t, Tcw tcw)   //Tcw - обобщенный параметр метода
        {
            Console.WriteLine(t.ToString() + tcw.ToString());
        }
    }

Т.е. новые фактические методы обобщенного метода определяются при их использовании, а не определении типа, а также 
обобщенные методы могут быть определены в необобщенном типе и наоборот. При этом и здесь все ссылочные типы будут
восприниматься одинаково.

Так же для удобства использования компилятор может сам подставлять типы в обобщенные аргументы-типы метода, это называется
механизмом выведения типов:
    private static void Main(string[] args)
    {
        int a = 10, b = 20;
        Swap(ref a, ref b); // вместо Swap<int>(ref a, ref b)
    }
    static void Swap<T>(ref T a, ref T b)
    {
        T c = a;
        b = a;
        a = c;
    }

Но при наличии необобщенного метода
