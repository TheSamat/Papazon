Если кто-то вместо обычного определения:
    List<DateTime> dt = new List<DateTime>();

Строет ракету, чтобы не писать <>:
    internal sealed class DateTimeList : List<DateTime>{}
    DateTimeList dt = new DateTimeList();

Бейте его по рукам! Типы DateTimeList и List<DateTime> совершенно разные:
    Boolean sameType = (typeof(List<DateTime>) == typeof(DateTimeList)); //false

Для уменьшения кодописания существуют синтаксические упрощения:
    using DateTimeList = System.Collections.Generic.List<System.DateTime>;

Здесь директория using определяет DateTimeList как тип System.DateTime и работает с ним соответственно:
    Boolean sameType = (typeof(List<DateTime>) == typeof(DateTimeList)); //true

Или использовать var:
    var dtl = List<DateTime>();
