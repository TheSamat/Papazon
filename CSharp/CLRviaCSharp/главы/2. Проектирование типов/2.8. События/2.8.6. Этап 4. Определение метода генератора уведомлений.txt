Этап 4. Определение метода, генерирующий события и отлавливающий определенные в коде ситуации

У класса-уведомителя должен быть и метод, генерирующий соответствующие события из входящей информации.

	public void SimulateNewMail(String from, String to, String subject) 
	{
		// Создать объект для хранения информации, которую нужно передать получателям уведомления
		NewMailEventArgs e = new NewMailEventArgs(from, to, subject);

		//Вызвать виртуальный метод, уведомляющий объект о событии
		//Если ни один из производных типов не переопределяет этот метод, объект уведомит всех зарегистрированных получателей уведомления
		OnNewMail(e);
		}
	}

Желательно в названии указывать имя события. В качестве аргументов принимать минимальное количество данных, при надобности 
проверить их и вычислить производные, а после передать данные в новый экземпляр типа-уведомления и вызвать метод-уведомитель, 
передав новое уведомление. Так подписчики получат 2 параметра: тип, который их уведомил (MailManager) и уведомление (NewMailEventArgs)

Сам по себе он вызываться не будет. Он будет вызываться в методах, действия которых определяет ситуацию при которой необходимо
будет создаваться событие.
