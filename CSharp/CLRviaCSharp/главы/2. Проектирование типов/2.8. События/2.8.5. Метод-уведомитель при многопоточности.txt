Метод-уведомитель может иметь 3 варианта реализации, отличающихся безопасностью:

Вариант 1: простой, но не защищен от асинхронности, при котором подписчиков не будет и вызовется исключение на Null
    protected virtual void OnNewMail(NewMailEventArgs e) 
    {
        NewMail?.Invoke(this, e)
    }
    
Вариант 2: проверка на null перед рассылкой, но не защищен от возможных изменений оптимизации компиляторов в будущем
    protected virtual void OnNewMail(NewMailEventArgs e) 
    {
      EventHandler<NewMailEventArgs> temp = NewMail;

      // А если подсчиков то удалили в ином потоке до вызова метода, исключения не будет
      // хоть это редкость, но подписка на стрим на ютубе, нарпимер, может многих бесить!
      if (temp != null) temp(this, e);
    }

Вариант 3: советуемый самим Рихтером, защищенный от всего, если вы параноик) - (с) CODE_BLOG
    protected void OnNewMail(NewMailEventArgs e) {
        EventHandler<NewMailEventArgs> temp = Thread.VolatileRead(ref NewMail);
        if (temp != null) temp(this, e);
    }

Дополнительно можно проверять уведомление на null:
        _ = e ?? throw new ArgumentNullException(nameof(e));
