Параметры или аргументы метдов - не то, чем кажется на первый взгляд. При передаче данных в метод по умолчанию передаются все переменные
по значению в виде нового обьекта с копированием всех значений в контексте вызываемого метода как значимых, так и ссылочных типов. При этом
для значимых типов идет копирование всех полей по их литералам, а ссылочных по ссылке. Из-за этого можно изменять поля ссылочных обьектов
передавая ссылочную переменную по значению, ибо поля изменяются по ссылке. Но изменять саму ссылку переданной переменной не имеет смысла,
ибо в методе изменится копия переменной (хоть это всего и ссылка), а не сама переданная переменная.

Для передачи же переменных по ссылке есть 3 оператора: ref, out и in, отличающихся только на этапе исходного кода и компиляции. На деле
имеют одинаковую сигнатуру для параметров, хоть и отличающихся от их отсутствия. Имеют следующие отличия:
	ref) передаваемая переменная должна быть инициализирована до вызова метода
	out) передаваемая переменная не обязана быть инициализирована до вызова метода, но должна быть инициализирована внутри метода
	in)  передаваемая переменная должна быть инициализирована до вызова метода, внутри же метода она приобретает свойство, доступное только
		 для чтения.

Использование передачи по ссылке ускоряет работу приложения, но не рекомендуется для большей читаемости и поддержки кода.
